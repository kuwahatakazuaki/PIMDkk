# PIMD8 日本語マニュアル

PIMD8 は、古典分子動力学（CLMD）と経路積分分子動力学（PIMD）を同じ入力形式で実行するプログラムです。このマニュアルは、PIMDを初めて使う利用者と、PIMD8の処理経路を確認したい開発者を対象にしています。

本書は現在のソースコードで動作が確認できる範囲だけを説明します。開発計画にだけ存在する機能や、実行経路まで接続されていない機能は扱いません。

## このマニュアルで使う用語

- **CLMD**: 古典分子動力学。`Isimulation=10`、`Nbead=1` で実行します。
- **PIMD**: 経路積分分子動力学。`Isimulation=0`、`Nbead>1` で実行します。
- **ビーズ**: PIMDで1個の原子を表す複数の複製。`Nbead` で数を指定します。
- **centroid**: 同じ原子に属する全ビーズの平均位置に対応するモードです。
- **backend**: 原子座標からポテンシャルエネルギーと力を計算する外部プログラムまたはモデルです。`Iforce` で選択します。
- **Resultディレクトリ**: 軌跡、エネルギー、restartなどを保存する場所です。
- **Scratchディレクトリ**: backendごとの一時入力・出力をビーズ別に保存する場所です。

## PIMDとCLMDの選択

| 目的 | `Isimulation` | `Nbead` | 備考 |
|---|---:|---:|---|
| 古典MD | `10` | `1` | 最初の動作確認に適しています |
| PIMD | `0` | `2`以上 | 既存例では `16` を使用しています |

`Isimulation=0`, `1`, `2` のいずれかを指定しても、`Nbead=1` なら入力読込時にCLMD（`Isimulation=10`）へ自動変更されます。意図を明確にするため、CLMDでは最初から `Isimulation=10` と書くことを推奨します。

## 最小実行の共通手順

PIMD8は実行時のカレントディレクトリにある `input.inp` を読みます。

```bash
cd 計算ディレクトリ
/path/to/PIMD8/pimd.exe
```

実行後は、まず `std.out` の末尾を確認します。

```text
***********************
    Simulation End!
***********************
```

次に `Result/ham.dat` と `Result/coor.xyz` が作成されていることを確認します。詳しくは[出力ファイル](outputs.md)を参照してください。

## 対象範囲

利用手順の中心は通常のPIMDとCLMDです。GaussianとMACEには実行チュートリアルがあります。VASPは既存コードの接続経路のみ[内部仕様](internals.md)で説明し、実行チュートリアルは設けません。

このマニュアルには、PIMDの理論導出、ビルド方法、外部プログラムのインストール方法、対応OS一覧は含めません。
